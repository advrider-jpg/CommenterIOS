# Release Risks

- Placeholder URLs and contact details must be replaced.
- Privacy answers must be rechecked against the final submitted binary.
- Screenshots are drafts and must be checked against real UI.
- Do not claim Australian Curriculum alignment without a separate content audit.
- Do not claim state approval, department approval, school approval, or universal compliance.
- If analytics, crash reporting, subscriptions, IAP, remote AI, cloud sync, accounts, or server upload are added later, update privacy copy and App Privacy answers.
- If iPad support is enabled, iPad screenshots may be required.
- If Languages becomes a claim, confirm dataset support first.
- The iCloud/device backup caveat must remain unless the app explicitly excludes data from backup.
- The public App Store name is metadata; the shorter home screen name is the on-device display name.
