# Release Proof Matrix

Submission readiness requires evidence files that are generated by real release
operations, not by local placeholders or prose claims.

Required proof artifacts:

- `Package.resolved` committed at the repository root after `swift package
  resolve` on a release machine.
- `docs/validation/archive-testflight-evidence.json` from a signed archive and
  TestFlight/App Store Connect validation run.
- `docs/validation/foundation-models-compile-evidence.json` from a current
  Xcode/iOS SDK lane that compiles the guarded Foundation Models source.
- `docs/validation/target-app-open-validation.json` from opening generated
  DOCX, XLSX, and XLS files in the declared target apps.

Run `python scripts/validate_release_proof_matrix.py` before treating the repo
as App Store submission-ready. Missing artifacts are expected during ordinary
development and are release blockers, not soft warnings.
